#
# FORMAT:
#
# description:
# - should use imperative tone: “change” not “changed” nor “changes”
# - don't capitalize first letter
# - no dot (.) at the end
#
# body:
# - should go into detail about changes made
# - bullet point or paragraph form are both acceptable
#
# footer:
# - should contain any issue tracking or pipeline actions or info
# - e.g. closes #123, #245, #992
# - e.g. [skip_ci]
#
